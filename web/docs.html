QUIVer Network ドキュメント

QUIVer Networkは、P2P技術を活用した分散型AI推論ネットワークです。 誰でもProviderとして参加し、余剰計算リソースを活用して収益を得ることができます。

📢 最新情報: v1.1.0がリリースされました。15種類以上のモデルに対応し、 ワンクリックインストールが可能になりました。

主な特徴

  • 完全分散型 - 中央サーバーに依存しないP2Pアーキテクチャ
  • 高速推論 - QUICプロトコルによる低遅延通信
  • 収益化 - 推論処理ごとに報酬を獲得
  • セキュア - Ed25519署名による改ざん防止
  • 柔軟性 - 15種類以上のAIモデルに対応

クイックスタート

1分でProviderになる

# macOS/Linux curl -fsSL https://quiver.network/install.sh | bash # 特定のモデルを指定する場合 export QUIVER_MODEL="qwen3:7b" curl -fsSL https://quiver.network/install.sh | bash

APIを使用する

# 無料アカウントで試す curl -X POST https://api.quiver.network/v1/generate \ -H "Content-Type: application/json" \ -d '{ "prompt": "Hello, world!", "model": "llama3.2:3b", "max_tokens": 100 }'

推論API

POST /v1/generate

リクエストパラメータ

パラメータ 型 必須 説明
prompt string Yes 入力プロンプト（最大32KB）
model string Yes 使用するモデル名（例: "llama3.2:3b"）
max_tokens integer No 最大生成トークン数（デフォルト: 1000）
temperature float No 生成の多様性（0.0-2.0、デフォルト: 0.7）
stream boolean No ストリーミングレスポンス（デフォルト: false）

SDK

JavaScript/TypeScript

// インストール npm install @quiver/sdk // 使用例 import { QUIVerClient } from '@quiver/sdk'; const client = new QUIVerClient({ apiKey: 'sk_live_...', baseURL: 'https://api.quiver.network' }); const response = await client.generate({ prompt: 'Explain quantum computing', model: 'qwen3:7b', maxTokens: 500 }); console.log(response.completion);

Python

# インストール pip install quiver-sdk # 使用例 from quiver import QUIVerClient client = QUIVerClient( api_key='sk_live_...', base_url='https://api.quiver.network' ) response = client.generate( prompt='Explain quantum computing', model='qwen3:7b', max_tokens=500 ) print(response.completion)

Go

// インストール go get github.com/quiver/sdk-go // 使用例 package main import ( "fmt" "github.com/quiver/sdk-go" ) func main() { client := quiver.NewClient("sk_live_...") resp, err := client.Generate(&quiver.GenerateRequest{ Prompt: "Explain quantum computing", Model: "qwen3:7b", MaxTokens: 500, }) if err != nil { panic(err) } fmt.Println(resp.Completion) }

監視とメトリクス

Provider メトリクス

Providerは以下のエンドポイントでメトリクスを公開します：

# Prometheusメトリクス http://localhost:8091/metrics # 健全性チェック http://localhost:8091/health # 統計情報 http://localhost:8091/stats

主要メトリクス

  • quiver_requests_total - 総リクエスト数
  • quiver_request_duration_seconds - リクエスト処理時間
  • quiver_tokens_processed_total - 処理トークン数
  • quiver_active_connections - アクティブ接続数
  • quiver_receipts_issued_total - 発行レシート数
💡 ヒント: Grafanaダッシュボードテンプレートが monitoring/ ディレクトリに用意されています。

トラブルシューティング

よくある問題

Provider が起動しない

# ポートが使用中の場合 lsof -i :4002 kill -9 [PID] # 権限エラーの場合 sudo chown -R $USER:$USER ~/.quiver # ログを確認 journalctl -u quiver-provider -f

P2Pネットワークに接続できない

  • ファイアウォール設定を確認（ポート4001-4003を開放）
  • ブートストラップノードの状態を確認
  • NAT設定を確認（UPnPを有効化）

推論が遅い

  • より小さいモデルを使用する
  • GPUアクセラレーションを有効化
  • 同時リクエスト数を制限
⚠️ 注意: 問題が解決しない場合は、 Discord のサポートチャンネルでお問い合わせください。
